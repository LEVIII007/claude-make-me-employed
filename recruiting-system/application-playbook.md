# Application Playbook

Use this when moving from job discovery into actual application work.

## Before starting a batch

- Confirm the target jobs are still fresh, real, and worth applying to.
- Work inside the user's default Chrome session so existing sign-ins can be reused.
- Verify resume upload once on a real form before opening several applications.
- Prefer the strongest direct company forms first, especially the ones that do not require new account creation.

## Browser workflow

1. Reuse an already signed-in tab if one exists.
2. Open the company application form, not just the LinkedIn job detail page.
3. Fill one section at a time and re-check the page after each meaningful interaction.
4. Save progress early when the site supports it.
5. Keep blocked or partially completed forms open instead of abandoning them.

## Form-filling rules

- Treat repeated `Select One` dropdowns as separate controls. Many enterprise forms reuse identical labels.
- After every dropdown selection, re-read the page. Some portals rerender fields, change button ids, or auto-advance.
- Do not trust imported defaults blindly. Review contact info, work authorization, notice period, and demographic fields carefully.
- On review pages, scan for contradictions or obviously wrong autofill before submitting.
- If a form asks something new, save the answer to the answer bank after the user confirms it.

## Portal-specific notes

### Workday

- `Use My Last Application` can reduce typing, but it does not guarantee the resume or profile will carry over correctly.
- Dropdowns are especially fragile. Scope actions carefully and re-snapshot often.
- Review pages can reveal bad defaults that were not visible earlier.

### SuccessFactors

- If `Upload from Device` is blocked, enable `Allow access to file URLs` for the Codex Chrome extension before retrying resume upload.
- Resume upload can auto-parse employment and education, but do not trust the imported values blindly.
- Re-check top contact fields after resume parsing. Street address, city, postal code, and questionnaire answers may still be blank.
- Imported work history often needs cleanup for titles, company names, end dates, and industry dropdowns.
- Some SuccessFactors comboboxes are easier to complete by typing the value and pressing `Enter` than by clicking the option list.
- Save the draft before the final submit step. The success banner is a useful validation checkpoint.
- Saving the draft can collapse sections, so reopen only the sections that still need review.

### Account-gated portals

- If a site requires account creation or sign-in and the user is not already signed in, pause that application and move to easier wins.
- Resume import via Indeed, Dropbox, OneDrive, or similar should only be used when the user is already signed in there.

## Submission policy

- Never submit without final approval.
- Right before submission, verify:
- Resume is attached
- Contact information is correct
- Work authorization answers are correct
- Review page has no contradictory autofill
- If any of these fail, fix them before asking for the final go-ahead.
